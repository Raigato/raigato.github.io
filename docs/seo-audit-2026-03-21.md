The site has solid SEO fundamentals: Lighthouse reports **SEO 100/100** on both desktop and mobile, HTTPS, sitemap, robots.txt, canonical, JSON-LD, and Open Graph are all correctly configured. The main gaps are on-page structural issues (missing H1, empty alt text, heading hierarchy), a few metadata inconsistencies, and stale supporting files.

## Critical Issues

### 1. Missing H1 Tag
- **Severity:** Critical
- **Where:** `src/pages/index.astro`, `src/components/Hero.astro`
- **Problem:** There is no `<h1>` element anywhere on the page. The hero section displays the name and role via a JavaScript typewriter animation (lines injected by GSAP at runtime), which means this content is not present in the static HTML delivered to crawlers. The only `<h2>` on the page is in the Contact section ("LET'S BUILD SOMETHING GREAT TOGETHER."). All other sections use `<h3>` without any parent H1 or H2.
- **Fix:** Add a visible or visually-hidden `<h1>` in the hero section containing the primary keyword phrase, e.g. `<h1 class="sr-only">Gabriel Raymondou — Freelance Fullstack Developer</h1>`. This is the single highest-impact SEO change possible on this page.

### 2. Empty Alt Text on All Project Images
- **Severity:** Critical
- **Where:** `src/components/ProjectCard.astro:39`
- **Problem:** Every project card image is rendered with `alt=""`. This means Google Image Search cannot index them, screen readers skip the images entirely, and accessibility is degraded. The attribute is hardcoded empty for all 4 projects.
- **Fix:** Add descriptive alt text sourced from the project data. In `src/data/projects.ts`, add an `alt` field to the `Project` interface (e.g. `"Screenshot of Growth Engineering Platform at Swile"`), then pass it to the `<img>` tag in `ProjectCard.astro`.

### 3. Title Tag / OG Title Inconsistency + Weak Keyword Targeting
- **Severity:** Critical
- **Where:** `src/pages/index.astro:20` (title) vs `:26` (og:title)
- **Problem:** The `<title>` tag reads "Gabriel Raymondou — Fullstack Developer" (39 chars — too short, missing "freelance"). The `og:title` reads "Gabriel Raymondou — Freelance Web & AI Developer" (48 chars). These two are inconsistent, and neither includes the word "freelance" in the `<title>` despite it being the primary conversion keyword. Google uses the `<title>` for SERP display, not og:title.
- **Fix:** Align both to a single keyword-optimised title, e.g. "Gabriel Raymondou — Freelance Fullstack Developer" (50 chars). Include "Freelance" near the front, keep under 60 characters.

## Medium Priority Issues

### 4. Color Contrast Failure (Lighthouse finding)
- **Severity:** Medium
- **Where:** `src/components/Hero.astro:52-53`
- **Problem:** The `[ HOVER ]` and `[ CLICK ]` hint labels use `text-white/20` (computed: `#3b3b3b` on `#0a0a0a` background). Lighthouse measured a contrast ratio of **1.76:1**, far below the WCAG AA minimum of 4.5:1. This is the only failing Lighthouse audit, costing 5 Accessibility points on desktop and 6 on mobile.
- **Fix:** Increase opacity to at least `text-white/50` (≈ #808080, contrast ~5.3:1), or consider removing these hint labels from the DOM and showing them only via CSS `::after` pseudo-elements which are skipped by accessibility tools since they are decorative.

### 5. Broken Heading Hierarchy
- **Severity:** Medium
- **Where:** `src/components/About.astro:11,15`, `src/components/ProjectCard.astro:17`, `src/components/Experience.astro:13`
- **Problem:** The page jumps directly from no H1 to multiple `<h3>` elements. Section labels like "Bio", "Stack", project titles, and experience titles are all `<h3>` with no H1 or H2 parent. This violates both WCAG 1.3.1 and Google's documented preference for logical heading structure.
- **Fix:** After adding the H1 (issue #1), add `<h2>` section headings for each major section. These can be visually hidden with `class="sr-only"` to preserve the design. Example: `<h2 class="sr-only">About</h2>` before the About section content.

### 6. Favicon Not Declared in HTML Head
- **Severity:** Medium
- **Where:** `src/pages/index.astro` (head section)
- **Problem:** `favicon.ico` and `favicon.svg` exist in `/public/` but there is no `<link rel="icon">` declaration in the HTML `<head>`. Browsers will auto-discover `/favicon.ico`, but without an explicit declaration there is no SVG favicon served to modern browsers, and no `apple-touch-icon` for iOS bookmarks.
- **Fix:** Add to the `<head>`:
  ```html
  <link rel="icon" href="/favicon.svg" type="image/svg+xml" />
  <link rel="icon" href="/favicon.ico" sizes="any" />
  ```

### 7. Social Links Missing `rel="me"`
- **Severity:** Medium
- **Where:** `src/components/Contact.astro:21,26,31`
- **Problem:** The GitHub, LinkedIn, and X social links do not include `rel="me"`. This attribute signals identity verification to search engines and is required for Google's Knowledge Panel and Mastodon/IndieWeb identity verification.
- **Fix:** Add `rel="me noopener noreferrer"` to all three social anchor tags. The existing `rel="noopener noreferrer"` just needs `me` appended.

### 8. llms.txt Outdated
- **Severity:** Medium
- **Where:** `public/llms.txt`
- **Problem:** The file lists three old projects (Studiz, Fliit Bot, HeroGrowth) and omits the three newest ones (Growth Engineering Platform, Casinoël, La Route des Golfs). It also references only 3 experience entries while the site shows 6. As AI crawlers (GPTBot, ClaudeBot, Googlebot) increasingly use this file for entity understanding, stale data leads to inaccurate AI-generated descriptions of the site owner.
- **Fix:** Update `llms.txt` to match the current `src/data/projects.ts` and `src/data/experiences.ts` content.

## Low Priority / Nice to Have

### 9. No `WebSite` Structured Data Schema
- **Severity:** Low
- **Where:** `src/pages/index.astro:41`
- **Problem:** Only a `Person` schema is present. Adding a `WebSite` schema with a `SearchAction` allows Google to display a site search box in SERP results. Adding a `ProfilePage` type alongside `Person` is recommended by Google for personal/portfolio sites.
- **Fix:** Add a second JSON-LD block with `@type: WebSite` including `name`, `url`, and optionally a `SearchAction`.

### 10. No `hreflang` for Regional Targeting
- **Severity:** Low
- **Where:** `src/pages/index.astro` (head section)
- **Problem:** Gabriel is based in France and targets both French and international clients, but there is no `hreflang` attribute. Without it, Google may not reliably serve the page to French-speaking audiences searching in French.
- **Fix:** If a French version is not planned, at minimum add `<link rel="alternate" hreflang="x-default" href="https://raigato.github.io/" />` to signal the default language explicitly.

### 11. Projects Section Missing `aria-label`
- **Severity:** Low
- **Where:** `src/components/Projects.astro:6`
- **Problem:** The About (`aria-label="About"`), Experience (`aria-label="Experience"`), and Contact (`aria-label="Contact"`) sections all have aria-labels, but the Projects section does not. This creates an inconsistency in landmark navigation for screen reader users.
- **Fix:** Add `aria-label="Selected Work"` to the `<section id="projects">` tag.

### 12. Orphaned Image Files
- **Severity:** Low
- **Where:** `public/images/projects/fliitbot.jpg`, `public/images/projects/herogrowth.jpg`
- **Problem:** Two project thumbnail images remain in `/public/images/projects/` that are no longer referenced by any project in `src/data/projects.ts`. They increase build artifact size and can cause confusion.
- **Fix:** Delete both files or move them to an archive folder outside `/public/`.

### 13. Footer Link Lacks Descriptive Text
- **Severity:** Low
- **Where:** `src/components/Footer.astro:13`
- **Problem:** The footer link "DESIGNED & BUILT BY RAIGATO" links to the GitHub repository but has no `aria-label` explaining the destination. Screen readers will announce vague link text.
- **Fix:** Add `aria-label="View source on GitHub"` to the footer anchor tag.

### 14. Hero Terminal Content Not Crawlable
- **Severity:** Low
- **Where:** `src/components/Hero.astro:59-60` (JS-injected terminal lines)
- **Problem:** The hero displays key identity information (name, role, location, availability) via a JavaScript typewriter animation that inserts DOM nodes at runtime. Googlebot crawls JS, but this content is absent from the initial HTML payload and relies on GSAP execution. It is not a heading and carries no semantic weight.
- **Fix:** Partially addressed by issue #1 (adding a sr-only H1). Optionally, render the terminal lines as static HTML with `visibility: hidden` in the initial paint and reveal via GSAP, so the content is in the DOM from the start.

### Title Tag Analysis

| Metric | Value | Assessment |
|---|---|---|
| Current title | "Gabriel Raymondou — Fullstack Developer" | |
| Character count | 39 chars | Too short (ideal: 50–60) |
| Contains "freelance" | No | Missing primary keyword |

### Keyword Alignment Audit

| Keyword | `<title>` | `<meta description>` | H1 | JSON-LD |
|---|---|---|---|---|
| freelance | No | Yes | No (no H1) | jobTitle ✓ |
| fullstack developer | Yes | Yes | No | Yes |
| React | No | Yes | No | knowsAbout ✓ |
| Next.js | No | Yes | No | knowsAbout ✓ |
| Cannes / France | No | No | No | address ✓ |
| AI integrations | No | Yes | No | knowsAbout ✓ |

**Gap:** Location keywords ("Cannes", "France", "remote") only appear in JSON-LD. Adding them to the meta description or a static text element would strengthen local SEO signals.

### Competitor Gap (brief)

Typical high-ranking freelance developer portfolio pages also include:
- Explicit availability status in visible static text (not only JS-injected)
- Rate or pricing indication (drives more qualified leads)
- Specific client names or case study metrics in copy (trust signals)
- A blog or articles section (long-tail keyword surface area)
